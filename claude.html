Chapter 03

Meet Claude Code.

Claude Code is a terminal-based AI coding agent. It can read your project, write new code, run commands, and debug errors, all by chatting with it. Think of it as a senior engineer willing to pair-program with you at 2 a.m.

Step 1

Install globally

One npm command and you're in.

Terminal 
npm install -g @anthropic-ai/claude-code
claude --version
Step 2

Authenticate

Run claude in any folder. It'll pop open a browser for you to log in with your Anthropic account.

Terminal 
cd my-first-app
claude
Step 3

Talk to it

Type like you're Slacking a coworker. No magic phrases, no syntax. Just intent.

Add a "dark mode" toggle to the top right of the page
C I'll add a button that toggles a data-theme attribute on the root element and persist the choice in localStorage. Starting now…

Vibe-coding: how to prompt like a pro

  • Describe outcomes, not code. "Make the submit button disabled until all fields are filled" is better than "use useState with a ref and a useEffect…"
  • Include the why. Claude writes better code when it knows the user story. "So our free-tier users don't accidentally spam the API" gives it context.
  • One thing at a time. Land a feature, test it, commit, then ask for the next one. Big-bang prompts land big-bang bugs.
  • Ask it to explain before it changes things you don't understand. "Before you edit, walk me through how auth works in this file" is a totally valid prompt.
  • Keep a rolling CLAUDE.md. Drop design decisions, library preferences, and "never do X" rules into CLAUDE.md. Claude reads it automatically.
  • Commit often. After every working change: git add . && git commit -m "...". If Claude breaks something, git restore . brings you back.

A good prompt, a bad prompt

Too vague

"Make it better."

Claude will guess. You'll hate the guess. You'll spend ten minutes undoing it.

Clear & contextual

"On the journal page, add a mood tag dropdown next to the save button with 5 options: happy, sad, anxious, calm, excited. Save it alongside the journal entry in localStorage."

Now Claude has a target. One try, working feature.
