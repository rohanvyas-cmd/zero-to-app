Chapter 07 · Guest field notes

Ten lessons from shipping AI in 48 hours.

Built an end-to-end AI productivity tool in a hackathon. The biggest wins weren't the framework or the UI. They were the system decisions underneath: routing, caching, prompt order, memory, feedback, security.

AI Systems Architecture Cost Memory Security
01
Architecture · Routing

Match each call to the task's real complexity.

Of 8 Claude calls in the pipeline, half moved to Haiku the moment the task turned mechanical. Don't default to Sonnet for everything. Use the right model for the right job.

Sonnet · Creative
4/8
High judgment

Drafting, synthesis, anything that benefits from nuance.

Haiku · Utility
4/8
Fast · Mechanical

Summarization, contradiction checks, pattern analysis.

02
Cost · Cheap calls

Utility calls don't need the flagship.

75%
cheaper

Routing utility calls to Haiku cost 75% less, with no visible drop in output quality. Sonnet for thinking, Haiku for checking.

Sonnet for thinking Haiku for checking 75% cheaper Zero quality drop
03
Preprocessing · Funnel

Filter aggressively before the model sees anything.

Simple Python ranking and de-duplication reduced 3,500 signals to 50 before Claude saw a single token. Pre-filtering is free. Inference isn't.

3,500
raw signals entering the system
↓   TF-IDF ranking · pure Python · zero tokens
~400
filtered candidates
↓   de-dupe & threshold
50
sent to Claude
04
Caching · Token economy

Cache the parts of your prompt that don't change.

90%
cost drop on repeated input

Guardrails, personality, system instructions: all stable across calls. Caching these dropped repeated input cost by 90%. The biggest single cost win of the build.

Guardrails cached Personality cached ~90% cost drop Zero resends
05
Prompt Order · Priority stack

The sequence decides what gets overridden.

Later parts of the prompt can override earlier ones. So put the non-negotiables first, and the flavor last.

p1
Guardrails
Hard constraints. Never overridden.
TOP
p2
Source content
The actual input being processed.
NEXT
p3
Feedback learnings
Corrections from past runs.
NEXT
p4
Personality
Tone and style. Last, so it can't override the rules above.
LAST
06
Feedback · The loop

Capture each run as a snapshot. Inject it into the next.

Snapshots stay minimal and token-budgeted: persistent enough to steer the next run, small enough not to bloat the prompt.

07
Memory · Layers

Short-term for consistency, long-term for identity.

Short-term

Consistency across recent outputs

Prevents contradictions between adjacent drafts. Flushes often.

Long-term

Behavior updates over time

Shapes identity and taste. Refines, never replaces. The system gets sharper over time, not different.

One horizon for the page in front of you, another for who the system is becoming.

08–10
Systems · Review · Humans

Three more things that mattered.

08

Security is a backend decision.

Secrets in .env on the server. API keys never touched the frontend. OAuth stayed server-side. Client code never saw credentials.

09

AI review only works with narrow asks.

"review this code" produced vague output. Specific asks (find leaks, dead code, inconsistencies) produced fixes I actually used.

10

Human review creates the real product.

The useful flow: extract → validate → score → human review. AI did most of the work. Final judgment stayed fast and manual.

Closing thought · 10 / 10
The best AI systems aren't prompt hacks. They're tightly designed workflows, with cost, memory, feedback, and security built in.
