Chapter 01

Concepts first.

Before we install anything, let's demystify the three words you'll hear constantly. If you understand these, 80% of engineering conversations suddenly make sense.

Frontend

The part the user sees and clicks on.

Anything rendered in a browser — buttons, forms, animations, layouts. Built with HTML, CSS and JavaScript, usually wrapped in a framework like React.

yourapp.com
Click me

Backend

The part that thinks and decides.

A server running code, living far away in a data center. It handles logic, auth, payments, AI calls — everything the user shouldn't see. Exposes its work through an API.

api.yourapp.com

Database

The place facts live permanently.

Users, posts, orders, messages — anything that must survive a page refresh. Think of it as a giant, well-organized spreadsheet the backend reads and writes to.

Every modern app — from Notion to Uber — is some flavor of this triangle.

Bonus vocabulary you'll hear

  • API — a menu of things a backend can do. "GET /orders" means "fetch orders."
  • JSON — the format data travels in. Looks like { "name": "Rohan" }.
  • Repository (repo) — a folder of code, tracked over time. Lives on GitHub.
  • Environment variable — a secret (API key, password) the app reads at runtime but doesn't commit to Git.
  • Framework — pre-built scaffolding for your app (React, Next.js, Express). Saves you from reinventing wheels.
