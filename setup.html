Chapter 02

Setting up your machine.

Copy, paste, press enter. Each tool below is one line of terminal magic. You'll use Mac's built-in Terminal app. Just open Spotlight (⌘ + space) and type "terminal".

01

Homebrew

What: The package manager for macOS.

Why: Installs everything else in this list with one command.

macOS
Terminal · install 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Terminal · verify 
brew --version
Windows user? Use Chocolatey or install WSL and follow Linux instructions. Everything after Homebrew works the same.
02

Git

What: The version-control system that tracks every change to your code.

Why: It's how you save snapshots, undo mistakes, and collaborate. It's also what GitHub is built on.

Terminal · install 
brew install git
git --version
Tell Git who you are

Every commit you make gets signed with these. Use your real name and the email tied to your GitHub account.

Terminal 
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
Run git config --global --list to double-check your identity is set correctly.
03

GitHub

What: Cloud storage and collaboration for Git repositories.

Why: Every deploy platform (Vercel, Railway) reads from GitHub. Your code lives here.

Step A · Create a free account
Open in browser 
# Sign up with the same email you gave Git above
open https://github.com/signup
Or go directly
github.com/signup
Step B · Install the GitHub CLI

The gh command lets you create repos, open PRs, and log in, all from the terminal.

Terminal 
brew install gh
gh --version
Step C · Authenticate once, forever

Walks you through logging in via your browser and sets up SSH keys automatically. No more typing passwords on git push.

Terminal 
gh auth login
Pick: GitHub.com → HTTPS → Yes, authenticate Git → Login with a web browser. Paste the one-time code, confirm, done.
Step D · Your first repo (one command)
Terminal 
mkdir hello-world && cd hello-world
echo "# hello-world" > README.md
git init
git add README.md
git commit -m "first commit"
gh repo create hello-world --public --source=. --push
Open https://github.com/<your-username>/hello-world. Your code is live on the internet.
04

Python

What: A programming language beloved by AI and data folks.

Why: Most AI libraries and scripts you'll touch are Python-first.

Terminal 
brew install python@3.12
python3 --version
05

Node.js + npm

What: JavaScript that runs outside the browser, plus its package installer.

Why: Every modern frontend tool (Vite, React, Claude Code) runs on it.

Terminal · nvm (recommended) 
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install --lts
node -v && npm -v
Terminal · brew (quicker) 
brew install node
06

Java

What: A battle-tested language and runtime.

Why: Some backend frameworks and enterprise tools still need it installed.

Terminal 
brew install openjdk@21
java --version
07

Vite

What: The blazing-fast scaffolder that creates modern web apps.

Why: We'll use it in Chapter 05 to spin up a React app in 10 seconds.

Terminal 
npm create vite@latest my-first-app -- --template react
cd my-first-app
npm install
npm run dev
Visit http://localhost:5173. You should see a spinning Vite logo. Congrats, you're running a React app.
08

VS Code

What: The editor where code lives and breathes.

Why: It's free, beautiful, and has the best AI extensions.

Install via brew 
brew install --cask visual-studio-code
Or download
code.visualstudio.com
Recommended extensions
Prettier
Auto-formats code so it looks professional.
ESLint
Catches bugs before you run the code.
GitLens
Shows who wrote each line and why.
Claude Dev ★
AI pair programmer, right inside the editor.
You've got a real developer machine now. Next up: let's introduce you to your new favorite coworker: Claude Code.
