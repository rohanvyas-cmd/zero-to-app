Chapter 04

Deployment tools, decoded.

Writing code on your laptop is step one. Getting a real URL that friends, customers, and investors can open is shipping. Here are the three platforms that make shipping embarrassingly easy.

Vercel

Frontend · one-click deploys

Connect your GitHub repo, push code, and Vercel builds and ships a live URL in under 60 seconds. Perfect for anything Vite, React, or Next.js.

Use it when
You have a frontend (React, Vue, etc.) and want it live on the internet without thinking about servers.
①Push code to GitHub
②Click "Import" on Vercel
③Ship.

Railway

Backend · APIs, databases, background jobs

Deploy servers, workers, and Postgres databases without touching AWS. Feels like Vercel, but for the backend half of your app.

Use it when
You need to run Python or Node code on a server: an API, a cron job, a Discord bot, anything that isn't a static website.
①New project → Deploy from repo
②Set env vars
③Railway gives you a live API URL

Supabase

Database + Auth + Storage

An open-source Firebase. A Postgres database, user authentication, and file storage, with a friendly dashboard and a one-line JavaScript client.

Use it when
You want users, login, and persistent data, without writing a backend at all.
①Create a project
②Copy your project URL + anon key
③Drop the client into your React app
A sensible default stack

A real, production-ready app. No backend code required.

"But which one do I use for my app?"

  • Just a website? Vercel is all you need.
  • Website + user logins + saved data? Vercel + Supabase.
  • Something that runs in the background (emails, cron jobs, AI pipelines)? Add Railway.
  • Enterprise or weird networking needs? AWS or GCP, but don't start there on day one.
